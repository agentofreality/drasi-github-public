# Agentic Workflows

This folder contains GitHub Agentic Workflow definitions (`.md` files) for the Drasi project.

These workflows use [GitHub Agentic Workflows (gh-aw)](https://github.github.io/gh-aw/) to run Copilot-powered agents in response to GitHub events.

## Available Workflows

| Workflow | Trigger | Description |
|---|---|---|
| `drasi-issue-researcher.md` | `needs-research` label on an issue | Produces a structured Research Brief comment analyzing the issue, identifying relevant code, proposing approaches, and listing open questions. |

## How to Use in a Repository

Agentic workflows **cannot** be inherited or called across repos. Each repository that wants to use a workflow must have its own copy of the `.md` and compiled `.lock.yml`.

### Prerequisites

- [gh CLI](https://cli.github.com/) installed
- [gh-aw extension](https://github.github.io/gh-aw/) installed: `gh extension install github/gh-aw`
- `COPILOT_GITHUB_TOKEN` org secret configured

### Steps

1. **Copy the `.md` file** into your repo's `.github/workflows/` directory:

   ```bash
   # From within your target repo (e.g., drasi-core)
   curl -o .github/workflows/drasi-issue-researcher.md \
     https://raw.githubusercontent.com/drasi-project/.github/main/agentic-workflows/drasi-issue-researcher.md
   ```

2. **Compile** the `.md` into a `.lock.yml`:

   ```bash
   gh aw compile
   ```

   This generates `.github/workflows/drasi-issue-researcher.lock.yml`.

3. **Commit and push** both files:

   ```bash
   git add .github/workflows/drasi-issue-researcher.md \
           .github/workflows/drasi-issue-researcher.lock.yml
   git commit -m "Add issue researcher agentic workflow"
   git push
   ```

4. **Test** by adding the `needs-research` label to any issue in your repo.

### Keeping in Sync

When the canonical `.md` in this folder is updated, each repo using it should re-copy and recompile:

```bash
cd your-repo
curl -o .github/workflows/drasi-issue-researcher.md \
  https://raw.githubusercontent.com/drasi-project/.github/main/agentic-workflows/drasi-issue-researcher.md
gh aw compile
git add .github/workflows/drasi-issue-researcher.md \
        .github/workflows/drasi-issue-researcher.lock.yml
git commit -m "Sync issue researcher workflow"
git push
```

## Modifying a Workflow

1. Edit the `.md` file in this folder (the source of truth).
2. Open a PR to `drasi-project/.github`.
3. After merge, notify repo maintainers to re-sync (see above).

> **Note:** Do not edit `.lock.yml` files directly — they are auto-generated by `gh aw compile`.
